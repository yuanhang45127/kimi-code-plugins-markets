Start a TDD cycle for: $ARGUMENTS

Follow the tdd-workflow skill:
1. **Red**: Write a failing test first — understand the feature, express the desired behaviour as a test
2. **Green**: Write minimal implementation to make the test pass
3. **Refactor**: Clean up while keeping tests green

Explain each phase before executing it. Confirm the test status after each phase.
